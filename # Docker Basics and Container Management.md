# Docker Basics and Container Management

## Objective

The objective of this task was to understand the fundamentals of Docker, learn the difference between Containers and Virtual Machines (VMs), and gain hands-on experience with Docker CLI commands for managing container lifecycles.

---

# Introduction

Docker is a containerization platform that enables developers to package applications along with their dependencies into lightweight, portable containers. Containers provide a consistent environment across different systems, making application deployment easier and more reliable.

---

# Containers vs Virtual Machines

| Feature        | Containers           | Virtual Machines                   |
| -------------- | -------------------- | ---------------------------------- |
| Architecture   | Share host OS kernel | Include a full guest OS            |
| Size           | Lightweight          | Heavyweight                        |
| Startup Time   | Seconds              | Minutes                            |
| Resource Usage | Low                  | High                               |
| Performance    | Near-native          | Additional virtualization overhead |
| Portability    | High                 | Moderate                           |

### Key Difference

A Virtual Machine emulates an entire computer system, including its own operating system. In contrast, a container packages only the application and its dependencies while sharing the host operating system kernel.

# Docker Concepts Learned

## Docker Image

A Docker image is a read-only template that contains the application code, libraries, dependencies, and configuration required to run an application.

Example:

```bash
docker pull nginx
```

## Docker Container

A container is a running instance of a Docker image. Containers are isolated, lightweight, and portable.

Relationship:

```text
Docker Image → Docker Container
```

## Docker Hub

Docker Hub is a cloud-based repository used to store and distribute Docker images.

---

# Practical Activities Performed

## 1. Pulling an Image

Downloaded the Nginx image from Docker Hub.

```bash
docker pull nginx
```

### Outcome

Successfully downloaded the Nginx image to the local system.

---

## 2. Running a Container

Started a container from the Nginx image.

```bash
docker run -d -p 8080:80 nginx
```

### Outcome

A running Nginx web server was created and made accessible through the browser using:

```text
http://localhost:8080
```

---

## 3. Viewing Running Containers

Checked active containers.

```bash
docker ps
```

### Outcome

Displayed details such as:

* Container ID
* Image Name
* Status
* Ports

---

## 4. Viewing Container Logs

Examined container logs.

```bash
docker logs <container_id>
```

### Outcome

Observed runtime information and server activity generated by the container.

---

## 5. Inspecting Container Details

Retrieved detailed container configuration.

```bash
docker inspect <container_id>
```

### Outcome

Viewed networking information, environment variables, mounted volumes, and container metadata.

---

## 6. Stopping a Container

Stopped the running container.

```bash
docker stop <container_id>
```

### Outcome

Container execution was successfully halted.

---

## 7. Restarting a Container

Restarted the container.

```bash
docker restart <container_id>
```

### Outcome

The container resumed operation without recreating it.

---

## 8. Removing a Container

Deleted the container from the system.

```bash
docker rm <container_id>
```

### Outcome

The container was permanently removed.

# Key Learnings

During this task, the following concepts were learned:

1. Docker is a containerization platform used to package and deploy applications.
2. Containers are lightweight and share the host operating system kernel.
3. Virtual Machines require a complete guest operating system and consume more resources.
4. Docker Images serve as templates for creating containers.
5. Docker Containers are running instances of Docker Images.
6. Docker Hub provides a centralized repository for Docker images.
7. Docker CLI enables management of images and containers through terminal commands.
8. Container lifecycle management includes:

   * Creating
   * Running
   * Inspecting
   * Stopping
   * Restarting
   * Removing containers
